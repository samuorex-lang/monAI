🧠 Reactive Architecture

Keyed State Boundaries

Moon leverages Effect's runtime to maintain clean, isolated sessions with reactive scoping.

Isolation by Design

Every session in Moon runs within an Effect scope. When a session ends, all resources — file watchers, agent processes, subscriptions — are automatically cleaned up.

SolidJS keyed primitives ensure that reactive state is properly scoped and destroyed, preventing memory leaks and stale references.

  • ✓ Automatic resource cleanup via Effect scopes
  • ✓ Keyed reactive state boundaries (SolidJS)
  • ✓ Isolated file context per session
  • ✓ Sandbox mode for untrusted code execution
  • ✓ Graceful error recovery with typed effects
